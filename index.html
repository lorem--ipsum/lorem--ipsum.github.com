It's all about the code

Making things right, and double check they are

A good software is a software that has been taken care of. I want code to be tested, rock-solid and optimized to offer the best experience. Testing, measuring the tests coverage and always improving it, having a critical look on tests relevance, this what I do I believe it is what makes the difference between software and piece of code. But producing good code is not good enough. Being aware of the latest trends in web interfaces and ergonomy as well as in software design, I really enjoy making projects move towards new technologies, frameworks and languages.

But what about the user ?

Good experience comes from good design

Good design should always be easy to implement, and it should makes the code stick to the user's needs. Everybody is the user to someone else, and I know how satisfying it is to discover that the guy who created whatever I'm using right now just perfectly understands my needs. This satisfaction is what fuels me.
